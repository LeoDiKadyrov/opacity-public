# Многоуровневая модель сети (OSI и TCP/IP)

Вся сложность компьютерных сетей управляема только потому, что её разбили на слои. Идея простая: каждый уровень решает одну задачу и общается только с соседними уровнями. Физический уровень гоняет биты по кабелю и не знает ничего про HTTP. HTTP не знает ничего про кабель. Это и есть инкапсуляция — каждый уровень добавляет свой заголовок, следующий добавляет свой поверх.

ОSI придумали как эталон с 7 уровнями: физический, канальный, сетевой, транспортный, сеансовый, представления, прикладной. В реальности победил TCP/IP с 4 уровнями, потому что он был уже реализован и работал, пока OSI ещё обсуждали на комитетах. Сеансовый и уровень представления в TCP/IP просто слились с прикладным.

Практический смысл этого разделения — независимость изменений. Можно заменить Wi-Fi на Ethernet на физическом уровне, и TCP поверх этого продолжит работать без изменений. Можно добавить TLS между транспортным и прикладным уровнями — HTTP не знает, что что-то поменялось.

Ключевой момент, который часто путают: PDU (protocol data unit) называется по-разному на каждом уровне. На транспортном — сегмент (TCP) или датаграмма (UDP). На сетевом — пакет. На канальном — фрейм. На физическом — биты. Это не просто терминология, это разные структуры данных с разными заголовками.
