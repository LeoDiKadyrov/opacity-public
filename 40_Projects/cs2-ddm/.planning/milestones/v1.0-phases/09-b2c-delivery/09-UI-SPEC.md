# Phase 9 — UI Design Contract

> Visual and interaction contract for the self-contained HTML report and the Streamlit download integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Context:** Output is a single `.html` file with all CSS inline — no component library, no CDN. Design tokens are carried from the Djok brand system (`landing/style.css`). The only Streamlit surface is one `st.download_button` added to the existing Interpretation tab.

---

## Design System

| Property | Value |
|-|-|
| Tool | none — inline CSS in Python string |
| Preset | not applicable |
| Component library | none |
| Icon library | none — text labels only |
| Font (body) | Space Grotesk, Inter, system-ui, sans-serif |
| Font (numbers/mono) | JetBrains Mono, Consolas, monospace |
| Template engine | Python f-string or Jinja2 (Claude's discretion) |

**Registry safety:** Report has zero external dependencies. No CDN links, no `<link rel="stylesheet" href="...">`, no `<script src="...">` pointing to remote URLs. All CSS is inlined in a `<style>` block. All fonts are declared via `font-family` stack with full system fallbacks — no Google Fonts requests. Charts are base64-encoded PNG data URIs. This is the safety gate: the HTML file must open correctly with no network access.

Source: D-05 (CONTEXT.md) + D-06 (CONTEXT.md)

## Spacing Scale

Declared values (multiples of 4px only). Applied via CSS custom properties or inline `style` attributes in the generated HTML.

| Token | Value | Usage |
|-|-|-|
| xs | 4px | Inline gaps between label and value within a cell |
| sm | 8px | Padding inside metric cells, table cell padding |
| md | 16px | Card internal padding, section-internal spacing |
| lg | 24px | Gap between metric cards and tier table |
| xl | 32px | Gap between report sections (Interpretation / Charts / Raw data) |
| 2xl | 48px | Top/bottom margin for section headers |
| 3xl | 64px | Page-level top padding |

Exceptions: Touch targets not applicable (HTML file, no interactive elements except static links). Table row height minimum 36px for readability at default browser zoom.

## Typography

All sizes in px, declared as CSS `font-size`. Report renders at browser default (16px root). Numbers always in JetBrains Mono stack.

| Role | Size | Weight | Line Height | Font Stack |
|-|-|-|-|-|
| Body | 14px | 400 (regular) | 1.5 | Space Grotesk, Inter, system-ui, sans-serif |
| Label | 12px | 500 (medium) | 1.4 | Space Grotesk, Inter, system-ui, sans-serif |
| Heading (section) | 20px | 600 (semibold) | 1.2 | Space Grotesk, Inter, system-ui, sans-serif |
| Display (worst metric punchline) | 28px | 700 (bold) | 1.1 | Space Grotesk, Inter, system-ui, sans-serif |
| Mono number | 14px | 400 (regular) | 1.5 | JetBrains Mono, Consolas, monospace |

**Rule:** Any value that is a number (RT in ms, angle in degrees, kill rate %, tier thresholds) uses the mono number style. Section headers, labels, and drill text use Space Grotesk.

## Component Specs

### 1. Worst Metric Card

Position: first element in each engagement-type section of the Interpretation section, before the tier table.

Layout:
- Container: border `1px solid #e8b84b`, background `#16161d`, border-radius `4px`, padding `16px`
- Top label: `Your biggest opportunity` in `#e8b84b`, font-size `12px`, font-weight 500, uppercase letter-spacing `0.08em`
- Metric name: font-size `28px`, font-weight 700, color `#e0e0e8`, margin-top `4px`
- Tier badge: inline after metric name, font-size `12px`
- Gap line: `Gap vs {benchmark}: {gap:+.1f}` in text-muted `#7a7a90`, font-size `14px`, mono stack for the number
- Drill line: full drill text in `#e0e0e8`, font-size `14px`, margin-top `8px`

If `get_worst_metric()` returns None or tier is `n/a`: card is omitted entirely. No empty card placeholder.

### 2. Tier Table

One table per engagement type (peek / hold). Section header above each: "Peek engagements" / "Hold engagements" in heading style (20px, semibold).

Table structure:
- Full-width `<table>`, border-collapse `collapse`
- Column widths: Metric 25%, You 10%, Tier 12%, Gap 12%, vs Benchmark 12%, Drill 29%
- Header row: background `#0e0e12`, text `#7a7a90`, font-size `12px`, font-weight 500, uppercase, padding `8px`
- Data rows: alternating background — odd rows `#16161d`, even rows `#0e0e12`
- Row border-bottom: `1px solid #2a2a35`
- Number cells (You, Gap, vs Benchmark): mono font stack, font-size `14px`
- Tier cell: tier badge (see color table above), font-size `12px`, padding `4px 8px`
- Survivorship bias note: `<tr>` spanning all columns, text `#7a7a90`, font-size `12px`, padding `4px 8px`, appears immediately after each RT metric row

### 3. Chart Section

Section header: "Distributions" in heading style (20px, semibold).

Layout: charts stacked vertically, one per metric per engagement type. Each chart:
- Container: background `#16161d`, border `1px solid #2a2a35`, border-radius `4px`, padding `16px`, margin-bottom `16px`
- `<img>` tag with `src="data:image/png;base64,{b64}"`, width `100%`, max-width `720px`
- Caption below image: `#7a7a90`, font-size `12px`, font-style italic

Chart render spec (for `report_generator.py` implementer):
- Figure size: `(10, 4)` inches at `96 dpi`
- Background: `#0e0e12` (figure) + `#16161d` (axes)
- Text color: `#e0e0e8`
- Accent color for player marker line: `#e8b84b`
- Grid: `#2a2a35`, alpha `0.5`
- No drop shadows, no gradients on bars

### 4. Raw Data Table

Section header: "Raw Data" in heading style (20px, semibold).

Table structure:
- Full-width `<table>`, border-collapse `collapse`, font-size `13px`
- Column priority order (left to right): `match_id`, `engagement_type`, `moment_timestamp`, `rt_visible_to_hit_ms`, `rt_visible_to_aim_ms`, `rt_aim_to_hit_ms`, `crosshair_angle_at_t0_deg`, `player_velocity_at_t0_ups`, `enemy_velocity_at_t0_ups`, `kill_rate`, then remaining columns
- Numeric columns: mono font stack
- Header: same style as tier table header
- Row striping: same as tier table
- If row count > 100: include count caption "Showing all {n} engagements"

Source: D-07 (CONTEXT.md) — "Raw data — full engagements table (all moments, all columns)".

## Streamlit Integration

### Download Button Placement

Location: bottom of the Interpretation section in `app.py`, after the peek/hold `st.tabs()` block, before the next `st.header()`.

Conditions for rendering:
- SteamID in session state is valid (same guard as existing interpretation block)
- `_benchmark_players` is non-empty (same guard as existing block)
- Render unconditionally once those guards pass — do not gate on "has worst metric"

Implementation pattern:
```python
_html_bytes = report_generator.generate_html_report(
    player_steamid=_interp_player_sid,
    benchmark_steamid=_bm_sid,
    benchmark_name=_bm_selection.split(" (")[0],
    db_path=DB_PATH,
)
st.download_button(
    label="Download Report",
    data=_html_bytes,
    file_name=f"djok_report_{_interp_player_sid}.html",
    mime="text/html",
    help="Self-contained HTML file — open in any browser, no install required.",
)
```

The button uses Streamlit's default styling — no custom CSS needed. Its gold accent alignment with the Djok brand is achieved through the label text alone; do not attempt to override Streamlit button color (fragile, version-dependent).

Source: D-08 (CONTEXT.md).
