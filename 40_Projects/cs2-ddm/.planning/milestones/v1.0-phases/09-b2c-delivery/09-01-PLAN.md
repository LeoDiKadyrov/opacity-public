## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → report_generator | player_steamid and benchmark_steamid come from Streamlit session state (operator-entered) — not user-supplied directly |
| report_generator → analytics.db | Read-only SQL queries; no user input interpolated into SQL — parameterised queries only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Injection | _raw_data_html SQL query | mitigate | Use parameterised sqlite3 execute("... WHERE player_steamid = ?", (steamid,)) — never f-string into SQL |
| T-09-02 | Information Disclosure | HTML report contains full SteamID64 | accept | By design — operator controls who receives the file; SteamID is public CS2 identifier |
| T-09-03 | Tampering | External URL in generated HTML | mitigate | External URL regex gate in verify step; CSS inlined in <style> block; no CDN references |
</threat_model>
