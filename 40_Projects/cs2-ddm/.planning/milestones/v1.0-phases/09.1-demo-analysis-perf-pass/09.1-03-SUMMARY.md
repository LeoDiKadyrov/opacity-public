# Phase 09.1 Plan 03: Selective parse_ticks Around player_hurt Anchors — Summary

Replace whole-demo `parse_ticks(props)` with a windowed `parse_ticks(props, ticks=union)` driven by per-player `player_hurt` anchors, gated by `config.PARSE_TICKS_SELECTIVE`. Subsumes backlog Phase 999.1.

## What Changed

**config.py** — three new constants (placed after `T0_MIN_OFFSET_TICKS` for thematic locality):
- `PARSE_TICKS_SELECTIVE: bool = True` (D-02 fallback toggle — flip to False to revert without git revert)
- `_SELECTIVE_WINDOW_BEFORE_TICKS: int = 384` (~6s pre-anchor)
- `_SELECTIVE_WINDOW_AFTER_TICKS: int = 192` (~3s post-anchor)

**ddm_analyzer.py**:
1. Added `import config` (module form) alongside the existing `from config import ...` block — required so `monkeypatch.setattr("config.PARSE_TICKS_SELECTIVE", ...)` is reflected at call time.
2. Imported the two window constants from config.
3. Added `DDMAnalyzer._build_tick_window_union(self, anchor_ticks: List[int]) -> List[int]` — vectorized `np.unique(np.concatenate([np.arange(t-384, t+192) for t in anchors]))` with `>= 0` clamp; empty input returns `[]`.
4. Added `DDMAnalyzer._parse_ticks_maybe_selective(self, props, anchor_ticks)` wrapper. Critical guard: `if PARSE_TICKS_SELECTIVE and anchor_ticks and len(window) > 0` — otherwise falls through to full parse with NO `ticks=` kwarg.
5. Reordered `analyze_demo()`: `parse_events([...])` block now runs FIRST (data-flow dependency — anchors must exist before window is built), `hurt_ticks` derived from `by_name["player_hurt"]` filtered to `attacker_steamid == self.player_steamid`, then `_parse_ticks_maybe_selective(all_props, anchor_ticks=hurt_ticks)` replaces the raw `pd.DataFrame(self.parser.parse_ticks(...))` call (both try and fallback exception paths).

## Deviations from Plan

**1. [Rule 3 — Blocking] Imported `config` as module in addition to existing `from config import ...`**
- **Found during:** Task 2 — re-reading SC3 RED tests showed they call `monkeypatch.setattr("config.PARSE_TICKS_SELECTIVE", False, raising=True)`.
- **Issue:** `from config import PARSE_TICKS_SELECTIVE` (the snippet shown verbatim in PLAN interfaces) snapshots the value at import time inside `ddm_analyzer.py`'s namespace. Subsequent `monkeypatch.setattr("config.PARSE_TICKS_SELECTIVE", False)` would not flip the flag the wrapper reads.
- **Fix:** Added `import config` and reference the flag as `config.PARSE_TICKS_SELECTIVE` inside `_parse_ticks_maybe_selective`. Window constants are still imported by name (they aren't monkeypatched in tests).
- **Files modified:** `ddm_analyzer.py`
- **Commit:** `b06fd6a`

**2. [Rule 3 — Blocking] Reordered `analyze_demo` body**
- **Found during:** Task 2.
- **Issue:** Existing code parsed `parse_ticks` BEFORE `parse_events`, but the wrapper needs `hurt_ticks` (from player_hurt) as input. Same dependency cited in RESEARCH §System Data Flow.
- **Fix:** Moved entire `parse_events` block (events_to_parse → all_hurt_df normalization) ABOVE the parse_ticks call site. Added a small block deriving `hurt_ticks` from `all_hurt_df`. Then the parse_ticks call (both try and except branches) is replaced with the wrapper.
- **Files modified:** `ddm_analyzer.py`
- **Commit:** `b06fd6a`

No other deviations. No auth gates. No checkpoints.
