## Decisions Made

- **Flag-existence assertions in tests** (not in plan, but necessary): the SC3 `disabled_full_parse` and `empty_anchors_falls_back_full` tests would have trivially PASSED on current code (production never sends `ticks=` kwarg, so "no ticks= kwarg" holds vacuously). To keep them honestly RED until plan 09.1-03 wires the flag, both tests now `assert hasattr(config, "PARSE_TICKS_SELECTIVE")` and `monkeypatch.setattr(..., raising=True)`. When 09.1-03 adds the flag, those asserts pass and the tests check actual behavior.
- **Subprocess runner inlined as string template** in `bench/run_bench.py` rather than a separate `_worker.py` — keeps the harness self-contained; only ~30 lines and clearly delimited by `_RUNNER_TEMPLATE`.
- **Empty `for_analysis/` accepted as baseline state**: the harness writes a non-empty JSON document with `n_demos: 0` and an explanatory `note` field. The `len(d) > 0` portion of the plan's verification command is satisfied; the `walltime_s` portion is documented as N/A until demos are available.
